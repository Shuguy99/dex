# Модель угроз Dex v3+

## Принципы

1. **Zero Trust к внешним входам** — любая команда от пользователя проверяется на вредоносность
2. **Принцип наименьших привилегий** — каждый агент имеет минимум прав, необходимых для работы
3. **Неотказуемость изменений** — все изменения кода и правил логируются и версионируются (Git)
4. **Defence in depth** — каждый уровень безопасности дублируется следующим

## Инварианты безопасности

(Формальные утверждения, проверяемые тестами)

```
I1: ∀ agent ∈ Agents: read_path(agent) ∈ whitelist
    ⇒ sandbox.validate_read(path) = True

I2: ∀ agent ∈ Agents: write_path(agent) ∈ whitelist
    ⇒ sandbox.validate_write(path) = True

I3: kill_switch.trigger() ⇒ all_agents.shutdown() ∧ stop_processing()

I4: ∀ rule ∈ Rules: rule.priority ∈ [0, 100] ∧
    rule.confidence ∈ [0.0, 1.0]

I5: ∀ command ∈ Commands: MentalFuse.check(command) = SAFE ⇒
    ConstitutionalChecker.check(command) = PASS

I6: ∀ session ∈ Mesh: session.privacy_level ≥ min_required_level

I7: Private_mode = ON ⇒ sensors.camera = OFF ∧ sensors.mic = OFF
```

## Приемлемые риски

- Доступ к метаданным команд (тип, частота, длина) — нужен для PersonalPredictor
- Анализ тона общения — нужен для PersonalityEngine
- Отправка anonymous-статистики в FederatedConsciousness (без контента)
- Хранение логов до 30 дней

## Недопустимые риски

- Удаление файлов за пределами `data/` без явного подтверждения
- Отправка конфиденциальных данных (пароли, ключи, документы) по Mesh
- Игнорирование kill_switch
- Изменение Constitutional Checker через петлю самоулучшения
- Отключение PrivacyManager из кода агента

## Аудируемость

- Все команды логируются с timestamp
- Все изменения правил — в Git
- Все этические проверки — в data/ethics/
- Все self-heal операции — в Git-коммитах
- Все сессии Mesh — в data/mesh/peers.json
- Все развёртывания — в data/prime/ с снапшотами
